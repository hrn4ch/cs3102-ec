Countability

Introduction

It is important to set up the background information necessary to begin understanding countability. When we say two sets are of the same size, we are saying they have the same cardinality. Sets can either hold a finite or infinite number of items, therefore, this cardinality may also be infinite or finite. Two sets can be related together by a function which can take an item from one set and return an item from the other depending on the regulations of this function.

Assume we have a function, f, for two sets, C and D such that:

f : C D
1-1, Injective
  • every element in the codomain is mapped to by at most one element in the domain
  • no item in D is mapped to by two items in C
  • |C| |D|
Onto, Surjective
  • every element in the codomain is mapped to by at least one element in the domain
  • every item in D is mapped to by at least one item in C
  • |C| |D|
f : C D
Bijective
  • both injective and surjective
  • every item in D is mapped to by exactly 1 item in C and vice versa
  • |C| |D|

The Pigeonhole Principle is when you have two sets, S and T, and a function, f, as follows:

f : S T

If |S| |T|, than f cannot be 1-1. This is because there are too many pigeons but not enough pigeonholes. There must be at least one hole (in T) shared by 2 pigeons (from S).

What Does It Mean For A Set To Be Countable?

A set is countable if:

  • It is finite
  • It has a bijection with the natural numbers (that means it is countably infinite)
  • It is a subset of a countable set

The union of countably many countable sets is countable.

Why Is It Important To Know If A Set Is Countable?

For anything to be computable by Java (or any other Turing-complete computing language) the set of possibilities must be countable. This means that the Java program must have a countable number of acceptable values in order to decide whether or not an inputted value is accepted.

Therefore, for us to code and use deciders within computer science, we have to be able to determine whether the acceptable items would be countable. Now, let’s look into how we determine whether or not a set is countable.

Methods For Determining Countability

If a set is known to be finite, we can conclude that it is a countable set. When a set is infinite, however, this must be proven a bit further. It is possible for infinite sets to be countable, like the natural numbers, ℕ, for example, but it is also possible for infinite sets to be uncountable, like all real numbers, ℝ.

Through finding a bijection, we are able to prove that there does exist a bijective function that maps the given set to the natural numbers. Through diagonalization, we are able to prove that there exists no possible bijection between the given set and the natural numbers.

TL;DR:

  • proving an infinite set is countable find a bijection with ℕ
  • proving an infinite set is not countable diagonalization

Finding A Bijection With The Natural Numbers

Finding a bijection is not always easy. Sometimes it is obvious, and sometimes it can be way more complicated. The goal of finding a bijection is making sure everything in the natural numbers can map both to and from an element of the set your looking at. If I am looking at a set like the set of all even numbers, it might be easier to just say multiply each natural number by 2 and you’ll get something in the set of even numbers. However, if you have the set of rational numbers, or the set of all possible strings over an alphabet, this is not the route you’d want to take.

Overall, if the infinite set you’re looking at seems like there might be an obvious function that can map between them, that is a definitive, acceptable proof of countability. If the set is more complicated, you might need to look into dovetailing.

Dovetailing is how you represent a bijection between a set and the natural numbers without actually having to know what that function is. With dovetailing, you can show that there exists a function that would be able to map any of the values in that set to the natural numbers by giving each cross-section an index.

To address the first issue, we can just assume the first rational number to be counted will be 0. Second, the bijective function represented by the dovetail is unknown, but if we state that the function is the result of the dovetail after having removed all duplicates, this issue is avoided.

Diagonalization

Now that we know how to prove if a set is countable, we should learn how to prove the opposite. If a set is uncountable it must also be true that I cannot list all values in the set infinitely, like we did for the first row and column of the dovetail. Now, to prove through diagonalization that a set isn't countable, we'll assume towards reaching a contradiction that it is countable and we can list all values in the set. For example, imagine we can find a bijection between the natural numbers and all real numbers that allows us to list the real numbers as such:

Source: http://steve-patterson.com/wp-content/uploads/2016/07/CantorDiagonalArgument.png

No matter how hard we try to list all real numbers, there will always be a number that we did not account for. Therefore, if I form a number by taking the first digit from the first number and adding one, and taking the second digit of the second number and adding one, and so on, I am able to form a number, x, which I could not have listed. There will always be a digit in x that is different from the one of the digits in the list all real numbers we made, meaning we couldn't have listed all real numbers. Therefore, we've reached a contradiction, and all real numbers are not countable.
